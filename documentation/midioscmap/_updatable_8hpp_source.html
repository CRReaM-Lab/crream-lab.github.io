45 // When a Derived instance is constructed, the base class constructor
46 // UpdatableCRTP is called first. Because the Derived constructor hasn't
47 // been run yet, the dynamic type is just Updatable, not yet Derived.
48 // The undefined behavior sanitizer checks this dynamic type when the this
49 // pointer is casted to Derived using the CRTP macro, and thus causes an
50 // error.
51 // The constructor only casts and stores the pointer, it doesn't dereference
52 // it to call any of Derived methods, so I don't think that it's undefined
53 // behavior (and I don't know of a better way to do this).
54 // Also see https://stackoverflow.com/q/61061051/6356744
